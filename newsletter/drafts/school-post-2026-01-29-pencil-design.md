## Le post

Claude Code ne fait plus seulement du code.

Il fait aussi du design.

J'ai découvert **Pencil**, un éditeur de design vectoriel qui s'intègre directement dans ton IDE (VS Code, Cursor) et qui se pilote avec Claude via MCP.

Et je dois avouer : j'ai passé des heures à jouer avec.

---

### Comment ça marche ?

Pencil, c'est un canvas infini de design vectoriel qui vit à côté de ton code. Les fichiers `.pen` sont versionnables avec Git comme n'importe quel fichier de code.

La magie : Claude a accès à cet éditeur via MCP. Tu lui décris ce que tu veux :
- "Crée-moi une landing page pour un SaaS B2B"
- "Fais un dashboard avec des KPIs"
- "Dessine un écran de connexion mobile"

Et Claude le fait. Pour de vrai.

Il peut :
- Créer des composants UI avec des bibliothèques intégrées (Shadcn UI, Halo, Lunaris...)
- Appliquer des styles via des variables et thèmes synchronisés avec ton CSS
- Générer des images avec l'IA
- Maintenir une synchronisation bidirectionnelle entre design et code

---

### Ce qui m'a surpris

1. **La qualité des outputs** - Ce n'est pas du wireframe basique. Avec les design systems intégrés (Shadcn UI notamment), les interfaces sont propres et utilisables.

2. **L'itération rapide** - Tu peux dire "change la couleur", "ajoute un bouton ici", "rends ça plus aéré" et il ajuste en temps réel.

3. **La cohérence** - Les variables et thèmes se synchronisent avec ton CSS. Un changement de couleur dans le design = un changement dans le code.

4. **L'intégration Git** - Les fichiers `.pen` se versionnent comme du code. Tu peux voir l'historique de tes designs, faire des branches, merger...

---

### À quoi ça me sert ?

- Prototyper rapidement des idées de SaaS
- Créer des mockups pour mes vidéos YouTube
- Visualiser une feature avant de coder
- Avoir une base pour discuter avec des clients
- Garder design et code synchronisés dans le même repo

Le fait que tout soit dans l'IDE change tout. Plus d'allers-retours entre Figma et VS Code. Tu designs, tu codes, tout au même endroit.
