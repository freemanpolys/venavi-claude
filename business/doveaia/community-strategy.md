## 2. DoveAIA Labs — Le concept

### 2.1 La proposition de valeur

> **"Les coulisses d'un entrepreneur qui construit des agents IA. Apprends avec moi, découvre mes erreurs, vole mes idées."**

**Ce que tu montres** :
- Tes projets perso (agents que tu construis)
- Tes expérimentations avec Claude Code, Azure AI, MCP
- Tes processus (comment tu travailles, tes outils)
- Tes apprentissages (ce que tu découvres chaque semaine)
- Tes revenus/métriques (si tu le souhaites, style Pieter Levels)

**Ce que tu NE montres PAS** :
- Les projets clients confidentiels
- Les informations sensibles de DoveAIA (pricing exact B2B, etc.)

### 2.2 Le contenu type

| Type | Exemple | Fréquence |
|------|---------|-----------|
| **Build log** | "J'ai créé un agent qui analyse mes emails en 3h" | 2/semaine |
| **Tutorial** | "Comment j'utilise Claude Code pour tout coder" | 1/semaine |
| **Fail story** | "Mon agent a coûté 200€ en une nuit (et comment l'éviter)" | 1/mois |
| **Tool review** | "J'ai testé 5 frameworks d'agents, voici mon verdict" | 1/mois |
| **Behind the scenes** | "Une journée dans ma vie de solopreneur IA" | 1/mois |
| **Veille** | "3 trucs IA de la semaine que tu dois connaître" | 1/semaine |

### 2.3 La différenciation vs BridgeMind

| Aspect | BridgeMind | DoveAIA Labs |
|--------|------------|--------------|
| Focus | Vibe coding (Cursor, tout) | Claude Code + Agents (Anthropic) |
| Ton | Américain, hype | Français, pragmatique |
| Angle | "Stop typing, start shipping" | "Je te montre comment je construis" |
| Communauté | Massive, généraliste | Niche, francophone, IA agents |
| Entreprise derrière | Non visible | DoveAIA visible (crédibilité) |

## 12. Système "Code → Contenu"

### 12.1 Le principe

```
Tu codes                    Tu captures                  Tu publies
─────────────────────────────────────────────────────────────────────
Session de travail    →    Notes + Screen record   →    Post ou vidéo
      (2-4h)                   (pendant)                 (5-10 min)
```

L'objectif : **capturer pendant que tu codes**, pas créer du contenu après coup.

### 12.2 Setup technique

**Dossier screen recordings**
```
~/Videos/build-logs/
```

**OBS — Configuration complète pour ne pas remplir le disque**

1. **Settings > Output > Recording**

| Paramètre | Valeur | Pourquoi |
|-----------|--------|----------|
| Type | Standard | Simple et fiable |
| Recording Path | `~/Videos/build-logs` | Centralisé |
| Recording Format | MKV | Anti-corruption si crash |
| Encoder | x264 (ou NVENC si GPU Nvidia) | CPU vs GPU |
| Rate Control | **CRF** | Qualité constante, taille variable |
| CRF | **28-30** | Plus haut = plus petit (23 = qualité, 30 = compact) |
| Preset | **veryfast** | Moins de CPU, fichiers un peu plus gros |

2. **Settings > Video**

| Paramètre | Valeur |
|-----------|--------|
| Base Resolution | 1920x1080 |
| Output Resolution | **1920x1080** (ou 1280x720 pour économiser) |
| FPS | **30** |

3. **Settings > Advanced > Recording**

| Paramètre | Valeur |
|-----------|--------|
| Filename Formatting | `%CCYY-%MM-%DD` |
| Automatically remux to mp4 | **Activé** |

**Espace disque estimé (avec CRF 28-30)**

| Qualité | Par heure | 1 semaine (40h) | 1 mois |
|---------|-----------|-----------------|--------|
| CRF 30 (compact) | ~200 MB | **8 GB** | 32 GB |
| CRF 28 (recommandé) | ~400 MB | **16 GB** | 64 GB |
| CRF 23 (haute qualité) | ~800 MB | 32 GB | 128 GB |

**Recommandation** : CRF 28, 1080p, 30fps → ~16 GB/semaine, qualité suffisante pour YouTube.

**Maintenance disque**
- Après édition/upload : supprimer les .mkv sources
- Garder 1 mois max de rushes non édités
- Script de nettoyage (optionnel) : supprimer les fichiers > 30 jours

**Facecam ?**
- **Début** : Écran seul (simple, moins de friction)
- **Plus tard** : Ajouter facecam pour YouTube (personal branding)

### 12.3 Workflow quotidien

**Début de session**
1. Ouvrir le fichier build log du jour
2. Lancer OBS (record en fond)
3. Noter l'objectif de la session

**Pendant le coding**
- Noter les découvertes, erreurs, solutions au fil de l'eau
- Parler à voix haute si tu veux (optionnel, aide pour les vidéos)

**Fin de session (5 min)**
1. Stopper OBS
2. Compléter le build log
3. Taguer si c'est "publiable" ou juste archive

### 12.4 Template Build Log

```markdown
# Build Log - [DATE]

## Objectif du jour
[Ce que tu voulais accomplir]

## Ce que j'ai fait
- [Action 1]
- [Action 2]

## Ce qui a marché ✅
- [Truc qui fonctionne bien]
- Prompt utile : "[le prompt]"

## Ce qui a merdé ❌
- [Problème rencontré]
- Solution : [comment tu l'as résolu]

## Découverte du jour 💡
[Un truc que tu ne savais pas avant]

## Temps passé
[X heures]

## Publiable ?
- [ ] Post LinkedIn/X
- [ ] Post Skool
- [ ] Vidéo YouTube (screen record intéressant)
- [ ] Juste archive
```

### 12.5 Transformation en contenu

| Source | Devient | Fréquence |
|--------|---------|-----------|
| Build log intéressant | Post LinkedIn/X | 2-3x/semaine |
| Build log + screen | Vidéo YouTube "Build log" | 1x/semaine |
| Plusieurs build logs | Newsletter récap | 1x/semaine |
| Découverte majeure | Thread X + post Skool | Quand pertinent |

>
> Le problème ? Les scopes.
> `gmail.full` → 403
> `gmail.readonly` → ✅
>
> Leçon : toujours demander le minimum de permissions.
>
> #buildinpublic #agentAI"
