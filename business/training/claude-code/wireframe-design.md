
# Wireframes fonctionnels — BizCraft

> Instructions pour un agent IA : générer les écrans, composants ou maquettes à partir de ce wireframe.

---

## 1️⃣ Page d’accueil — Proposition de valeur

**Objectif UX** : Faire comprendre rapidement la valeur de BizCraft et pousser vers l’audit gratuit.

**Sections** :
- **Header** : Logo BizCraft, menu (Méthode, Fonctionnalités, Pour qui ?, Tarifs), bouton principal “Faire le diagnostic gratuit”
- **Hero** : Titre + sous-titre + CTA principal “Faire le diagnostic gratuit” + CTA secondaire “Voir comment ça marche” + illustration (radar ou schéma)
- **Le problème** : 3 cards (ex. “Je fais tout moi-même”, “Rien n’est documenté”, “Impossible de déléguer”)
- **La solution BizCraft** : Timeline 4 étapes (Diagnostic → Roadmap → Process → Pilotage)
- **Preuve / crédibilité** : mention E-Myth, icônes méthode/système
- **Footer** : Mentions légales, RGPD, contact, disclaimer E-Myth

---

## 2️⃣ Onboarding — Setup entreprise

**Objectif UX** : Créer l’engagement et préparer la personnalisation.

**Stepper** : Entreprise → Objectifs → Maturité → Validation

**Étapes** :
1. Infos entreprise : nom, secteur, taille, pays → bouton Continuer
2. Objectifs : checkbox multiples (Déléguer, Standardiser, Vendre plus, Reprendre du temps)
3. Niveau de maturité : slider ou radio (Chaotique / Structuré / En croissance / Optimisé)
4. Validation : résumé + message → CTA “Accéder au Dashboard”

---

## 3️⃣ Dashboard — Progression & KPIs

**Objectif UX** : Vision claire et motivante.

**Layout** : 2 colonnes

- **Gauche — Roadmap** : checklist des étapes avec progress bar globale
- **Droite — KPIs** : cards (Process documentés, Rôles définis, Tâches déléguées, Niveau de dépendance)
- **Actions recommandées** : cards cliquables (ex. “Créer votre premier process”, “Définir votre rôle principal”)

---

## 4️⃣ Audit E-Myth — Diagnostic

**Objectif UX** : Questionnaire fluide et rapide, effet “test sérieux”.

**Page Questionnaire** :
- Header : progression (ex. Question 4/20)
- Question principale + 4 réponses radio / slider
- Bouton Suivant

**Résultats** :
- Radar graphique : Entrepreneur / Manager / Technicien
- Texte explicatif dynamique
- Bloc email : champ email + consentement RGPD + bouton “Recevoir mon rapport”

---

## 5️⃣ Builder de Process — Liste

**Objectif UX** : Simplifier la création de process et rendre l’outil addictif.

**Layout** : Sidebar gauche + contenu principal

- **Sidebar** : liste des process, bouton ➕ Nouveau process
- **Contenu principal** : table/cards (Nom, Type, Statut, Dernière mise à jour)

---

## 6️⃣ Page Process — Éditeur & checklist

**Objectif UX** : Édition et exécution claires.

**Header** : Nom du process + boutons Export PDF / Assigner / Versionner

**Steps** :
- Drag & drop
- Titre, Description
- Checklist items avec assignation + deadline
- Toggle Edition / Exécution

---

## 7️⃣ Documents — Templates & exports

**Objectif UX** : Montrer valeur premium et gain de temps.

**Tabs** : Rapports, SOPs, Légal, RH

**Cards** : Nom, description, boutons Générer / Télécharger PDF/Word

---

## 8️⃣ Team / Roles — Organigramme

**Objectif UX** : Visualiser l’entreprise comme un système.

- Organigramme visuel : rôles, responsabilités, process associés
- Actions : ➕ Nouveau rôle, lier un process à un rôle

---

## 9️⃣ Settings / Intégrations / Billing

**Tabs** : Profil, Entreprise, Intégrations, Sécurité, Facturation

**Intégrations** : Stripe, Notion, Google Drive, Webhooks

**Billing** : Plan actuel, historique factures, bouton upgrade/downgrade

---

## 🧠 Principes UX clés BizCraft

- Toujours montrer **l’étape suivante**
- Orienté **action**, pas contenu
- “Entreprise = système”
- Peu de texte, beaucoup de structure
- Freemium **sans frustration**
