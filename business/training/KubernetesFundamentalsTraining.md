## Training Objectives

By the end of this training, participants will be able to:

- Understand Kubernetes architecture and core concepts
- Deploy and manage containerized applications on a cluster
- Configure basic resources (Pods, Deployments, Services)
- Implement data persistence
- Apply security and monitoring best practices

---

## Detailed Program

### Day 1: Introduction and Architecture

**Morning (3h30)**
- Introduction to container orchestration
- Kubernetes history and ecosystem (CNCF)
- Kubernetes cluster architecture
  - Control Plane: API Server, etcd, Scheduler, Controller Manager
  - Worker Nodes: kubelet, kube-proxy, Container Runtime
- Setting up a development environment (Minikube, Kind)

**Afternoon (3h30)**
- Getting started with kubectl
- Pods: the basic unit of Kubernetes
- Labels, Selectors, and Annotations
- Namespaces and resource organization
- **Lab**: Deploy your first application

### Day 2: Workloads and Networking

**Morning (3h30)**
- ReplicaSets and Deployments
- Deployment strategies (Rolling Update, Recreate)
- DaemonSets and StatefulSets
- Jobs and CronJobs
- **Lab**: Update an application with zero-downtime

**Afternoon (3h30)**
- Kubernetes networking model
- Services: ClusterIP, NodePort, LoadBalancer
- Ingress and Ingress Controllers
- Internal DNS and service discovery
- **Lab**: Expose a multi-tier application

### Day 3: Configuration, Storage, and Observability

**Morning (3h30)**
- ConfigMaps: externalizing configuration
- Secrets: managing sensitive data
- Environment variables and configuration volumes
- Volumes and Persistent Volumes
- StorageClasses and dynamic provisioning
- **Lab**: Application with persistent database

**Afternoon (3h30)**
- Resource management (requests/limits)
- Probes: Liveness, Readiness, Startup
- Introduction to monitoring with Prometheus and Grafana
- Logs and debugging
- Best practices and next steps
- **Final Lab**: Complete deployment of a 3-tier application
