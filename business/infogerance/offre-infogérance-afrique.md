# Offre Exploitation, Maintenance & Support

**Applications Angular & Spring Boot — Serveurs / VMs Linux**

---

## Ce que nous faisons pour vous

Nous prenons en charge l'exploitation de votre infrastructure applicative pour que vous puissiez vous concentrer sur votre métier :

- **Déployer** les versions livrées par votre équipe de développement, de manière fiable et reproductible
- **Exploiter** vos serveurs et votre infrastructure au quotidien
- **Surveiller** la disponibilité et la santé de vos environnements
- **Intervenir** rapidement en cas d'incident d'infrastructure ou d'indisponibilité

## Classification des incidents

### Niveaux de criticité

| Niveau | Disponibilité | Temps de réponse | Temps de résolution |
|--------|---------------|------------------|---------------------|
| **P1 — Critique** | 24/7 (Premium) / Plage horaire (Basic, Standard) | 30 min (Premium) / 1h (Standard) / 4h (Basic) | 4h |
| **P2 — Majeur** | Jours ouvrés | 2h (Premium, Standard) / 4h (Basic) | 8h |
| **P3 — Mineur** | Jours ouvrés | 4h | 24h |
| **P4 — Demande** | Jours ouvrés | J+1 ouvré | 48h |

### Grille de qualification des incidents

| Niveau | Critères de qualification |
|--------|--------------------------|
| **P1 — Critique** | Application de production totalement indisponible, perte de données en cours, faille de sécurité active exploitée, impact sur 100% des utilisateurs |
| **P2 — Majeur** | Fonctionnalité majeure indisponible, dégradation sévère des performances (>50%), pas de contournement possible, impact sur >30% des utilisateurs |
| **P3 — Mineur** | Fonctionnalité secondaire impactée, contournement possible, dégradation légère des performances, impact limité |
| **P4 — Demande** | Demande d'information, changement de configuration non urgent, question technique, demande de déploiement planifié |

### Exemples de classification

| Situation | Niveau |
|-----------|--------|
| Serveur de production inaccessible, application KO | P1 |
| Base de données corrompue, perte de données | P1 |
| Faille de sécurité active (certificat compromis, intrusion) | P1 |
| API en erreur 500, front-end Angular inaccessible | P2 |
| Temps de réponse x3, dégradation sévère | P2 |
| Un environnement de staging indisponible | P3 |
| Certificat SSL expire dans 7 jours | P3 |
| Logs non collectés depuis 2h | P3 |
| Demande de déploiement d'une nouvelle version | P4 |
| Question sur la configuration serveur | P4 |

> **Clause de reclassification** : Nous nous réservons le droit de reclassifier un incident si les critères objectifs ne correspondent pas au niveau déclaré. En cas de reclassification, le client en est informé avec justification.

## Ce qui n'est pas inclus

Les prestations suivantes ne font pas partie de ces offres, quelle que soit la formule choisie :

- Correction de bugs applicatifs (responsabilité de votre équipe de développement)
- Développement de nouvelles fonctionnalités
- Mise à jour des dépendances applicatives (Spring Boot, Angular, librairies)
- Refonte d'architecture applicative
- Migration majeure d'infrastructure
- Ajout de nouveaux environnements serveur
- Intervention sur des composants tiers hors de notre périmètre

Ces prestations peuvent être proposées sur devis, en régie ou au forfait.

## Questions fréquentes

**Que se passe-t-il si je dépasse mon quota de déploiements ?**
Chaque déploiement supplémentaire est facturé unitairement :

| Offre | Tarif par déploiement supplémentaire |
|-------|--------------------------------------|
| Basic | 250 € HT |
| Standard | 250 € HT |
| Premium | 200 € HT |

**Puis-je changer d'offre en cours d'année ?**
Oui, vous pouvez passer à une offre supérieure à tout moment. Le complément est calculé au prorata.

**Qu'est-ce qui déclenche un ticket de support ?**
Tout incident d'infrastructure, indisponibilité ou dysfonctionnement sur les serveurs couverts. Les bugs applicatifs sont diagnostiqués et escaladés vers votre équipe de développement.

**L'astreinte 24/7, concrètement ?**
En offre Premium, un ingénieur est joignable à tout moment pour les incidents critiques d'infrastructure (serveur inaccessible, saturation disque, certificat expiré, faille de sécurité active).

**Qui corrige les bugs de l'application ?**
La correction de bugs relève de votre équipe de développement. Nous diagnostiquons l'incident (infra vs applicatif), et si le problème est applicatif, nous vous transmettons le diagnostic pour que votre équipe puisse intervenir.

**Je n'ai pas encore d'infrastructure en place, pouvez-vous la construire ?**
Oui. Nous proposons une offre BUILD dédiée à la mise en place de vos environnements serveur (de 2 à 5 serveurs, avec options haute disponibilité et observabilité). Consultez notre offre BUILD pour plus de détails.

**Comment se déroulent les interventions à distance ?**
Toutes les interventions sont réalisées à distance via accès sécurisé (SSH/VPN). Le décalage horaire entre la France et l'Afrique de l'Ouest (UTC vs UTC+1) est minimal et n'impacte pas la réactivité du support.
