# Brain Dump - 2026-01-20 16:18

Comment maintenir des produits open-source et des produits premiums utilisés par le Git Repo ?

Les 4 stratégies principales :

## Strategy 1: Private Fork (Plus courant)

public-repo (open source)     →    private-repo (premium)
      ↑                                    ↓
   community PRs                    premium features
                                            ↓
                               regularly merges from public

- Public repo contient la version open source
- Private repo est un fork qui ajoute les features premiums
- Merge régulier des changements public → private
- Code premium ne revient jamais dans le public

**Utilisé par**: GitLab, Grafana, Mattermost

## Strategy 2: Core + Plugins/Extensions

public-repo/
├── core/           (open source)
└── plugin-api/     (open source)

private-repo/
└── premium-plugins/  (premium features as plugins)

- Core product et plugin API sont open source
- Premium features sont des plugins séparés dans un repo privé
- Premium builds bundle core + premium plugins

**Utilisé par**: WordPress, IDEs

## Strategy 3: Single Repo avec Build-Time Feature Flags

repo/
├── src/
│   ├── features/
│   │   ├── basic/        (included in OSS)
│   │   └── premium/      (excluded from OSS builds)
│   └── ...
└── build configs

- Un repo privé avec tout le code
- Build process exclut le code premium pour les releases OSS
- **Caveat**: Ne marche pas si le repo est public

## Strategy 4: Monorepo avec Private Packages

monorepo/
├── packages/
│   ├── core/           (public, npm published)
│   ├── ui/             (public, npm published)
│   └── premium/        (private, never published)
└── apps/
    ├── oss-app/        (imports only public packages)
    └── premium-app/    (imports all packages)

- Monorepo privé contient tout
- Seuls les packages publics sont publiés sur npm/pkg registries

## Pour agentdx (Go CLI tool)

**Option A: Private Fork**

```bash
gh repo create yourcompany/agentdx-enterprise --private
git clone yourcompany/agentdx-enterprise
cd agentdx-enterprise
git remote add upstream https://github.com/yourcompany/agentdx.git

# Regular workflow: sync open source changes
git fetch upstream
git merge upstream/main
```

**Option B: Plugin Architecture**

```go
// In open source: define extension points
type PremiumFeatures interface {
    AdvancedSearch(query string) ([]Result, error)
    // ... premium capabilities
}

// Premium features register at build time
var premiumModule PremiumFeatures
```
